# Freedom Intelligent Routing Engine (FIRE)

Проект разработан в рамках хакатона **Datasaur 2026** для кейса от **Freedom Broker**.

##  Описание проекта
**FIRE** — это интеллектуальная система автоматической классификации и маршрутизации входящих заявок (тикетов) клиентов. Система призвана автоматизировать первичную обработку обращений, снизить нагрузку на операторов и повысить скорость обслуживания.

### Основные функции:
*   **Интеллектуальная классификация:** Автоматическое определение типа обращения (жалоба, запрос, благодарность и т.д.) с помощью LLM (Google Gemini).
*   **Суммаризация:** Выделение главной сути из длинных описаний проблем.
*   **Умная маршрутизация:** 
    *   Поиск ближайшего офиса на основе геоданных клиента и филиалов.
    *   Назначение наиболее подходящего менеджера (учитываются навыки, специализация и текущая нагрузка).
*   **Автоматическая обработка данных:** Импорт справочников и заявок из CSV-файлов.

##  Технологический стек
*   **Backend:** Python 3.10+, Django 5.x, Django REST Framework.
*   **Искусственный интеллект:** Google Gemini SDK (`google-genai`).
*   **Геолокация:** Geopy (расчет расстояний между клиентом и точками обслуживания).
*   **База данных:** SQLite (локально) / PostgreSQL (production).

##  Структура проекта
*   `classificator/` — Ядро бизнес-логики:
    *   `classificate.py` — Оркестрация процесса анализа.
    *   `get_office.py` — Расчет гео-близости и подбор офиса.
    *   `get_manager.py` — Логика выбора ответственного сотрудника.
    *   `summary.py` — Генерация кратких выжимок текста.
*   `datazavr/` — Django-приложение:
    *   `api/` — Модели, сериализаторы и API-вьюхи.
    *   `core/` — Глобальные настройки проекта.
    *   `data/` — Исходные CSV-данные для первоначальной загрузки.

##  Установка и запуск

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/Alim-Rakhmet/Freedom-Intelligent-Routing-Engine.git
   cd Freedom-Intelligent-Routing-Engine
   ```

2. **Настройте виртуальное окружение:**
   ```bash
   python -m venv venv
   source venv/bin/activate  # Для Windows: venv\Scripts\activate
   pip install -r requirements.txt
   ```

3. **Настройте переменные окружения:**
   Создайте файл `.env` в корневой директории или в папке `datazavr/` (в зависимости от вашей конфигурации) и укажите `GEMINI_API_KEY`.

4. **Выполните миграции и подготовьте БД:**
   ```bash
   cd datazavr
   python manage.py migrate
   ```

5. **Запустите сервер:**
   ```bash
   python manage.py runserver
   ```

При первом вызове API эндпоинта система автоматически загрузит данные из CSV файлов, находящихся в `datazavr/data/`, если база пуста.
